# Google Credentials Setup Guide

This project requires two separate Google credentials:

1. **Service Account** — for reading the Google Sheet (no user interaction needed)
2. **OAuth2 Client ID** — for sending emails via Gmail (requires one-time user consent)

Both are created in the same Google Cloud project.

---

## Step 1: Create a Google Cloud Project

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Click the project dropdown at the top and select **New Project**
3. Give it a name (e.g., "River Level Notifications") and click **Create**
4. Make sure the new project is selected in the dropdown

---

## Step 2: Enable the Required APIs

1. In the left sidebar, go to **APIs & Services > Library**
2. Search for and enable each of these:
   - **Gmail API**
   - **Google Sheets API**

---

## Step 3: Create a Service Account (for Google Sheets)

The service account lets the system read the subscriber spreadsheet without any user login.

1. Go to **IAM & Admin > Service Accounts**
2. Click **Create Service Account**
3. Give it a name (e.g., "river-notify-sheets") and click **Create and Continue**
4. Skip the optional role/access steps — click **Done**
5. Click on the service account you just created
6. Go to the **Keys** tab
7. Click **Add Key > Create new key**
8. Choose **JSON** and click **Create**
9. A `.json` file downloads — this is your `service_account.json`

**Important**: Share your Google Sheet with the service account's email address (it looks like `river-notify-sheets@your-project.iam.gserviceaccount.com`). Give it **Viewer** access. Without this step, the system can't read the sheet.

---

## Step 4: Configure the OAuth Consent Screen (for Gmail)

Before creating OAuth credentials, you need a consent screen configured.

1. Go to **Google Auth platform > Branding** (or **APIs & Services > OAuth consent screen** in older UI)
2. If prompted, click **Get Started**
3. Fill in:
   - **App name**: e.g., "River Level Notifications"
   - **User support email**: your email
4. Click **Next**
5. Under **Audience**, select **External** (unless you have a Google Workspace org, then use Internal)
6. Click **Next**
7. Add a contact email and click **Next**
8. Agree to the terms and click **Create**

If you chose "External", your app will be in "Testing" mode. That's fine — you just need to add your Gmail address as a test user:

1. Go to **Google Auth platform > Audience**
2. Under **Test users**, click **Add users**
3. Add your Gmail address

---

## Step 5: Create OAuth2 Client Credentials (for Gmail)

1. Go to **Google Auth platform > Clients** (or **APIs & Services > Credentials**)
2. Click **Create Client**
3. Set **Application type** to **Desktop app**
4. Give it a name (e.g., "River Notify Desktop")
5. Click **Create**
6. Click **Download JSON** — this is your `gmail_credentials.json` (also called "client secret" file)

---

## Step 6: Generate the Gmail Token (one-time)

Run the token generator utility that's part of this project:

```bash
python src/create_token.py
```

This will:
1. Open your browser to a Google sign-in page
2. Ask you to grant the app permission to send emails on your behalf
3. After you approve, it saves a `token.json` file locally

The `token.json` contains your access token and refresh token. The system uses this to send emails without asking you to log in again. The token auto-refreshes, so you should only need to do this once unless you revoke access.

## File Summary

| File | What it is | How you get it | Commit to git? |
|------|-----------|----------------|----------------|
| `service_account.json` | Service account key for Sheets | Download from Cloud Console (Step 3) | **NO** — contains private key |
| `gmail_credentials.json` | OAuth client ID/secret | Download from Cloud Console (Step 5) | **NO** — contains client secret |
| `token.json` | OAuth access + refresh token | Generated by running `create_token.py` (Step 6) | **NO** — contains auth tokens |

Add all three to your `.gitignore`.

## Troubleshooting

**"Access blocked: This app's request is invalid"**
- Make sure you've added your email as a test user (Step 4)

**"The caller does not have permission" when reading the sheet**
- Make sure you shared the Google Sheet with the service account email address (Step 3)

**"Token has been expired or revoked"**
- Re-run `python src/create_token.py` to generate a fresh token

**"Gmail API has not been enabled"**
- Go back to Step 2 and enable the Gmail API in your project
